Microsoft Fabric + ADLS Gen2

Hybrid Security Architecture — Visual Reference Documentation

Three-Layer Security Architecture

Each layer enforces independently. A user must pass all three checks to access data. Failure at any layer blocks access regardless of permissions at other layers.

Medallion Architecture: ADLS Gen2 Storage + Fabric Compute via Shortcuts

Separate ADLS Gen2 storage accounts per layer with dedicated Fabric workspaces connected via OneLake shortcuts. Each storage account has independent firewall rules and RBAC.

Control Plane vs Data Plane: Role Separation Model

Platform administration (Control Plane) and data product development (Data Plane) operate with distinct roles, permissions, and responsibilities.

Security Evaluation Flow: User Query to Data Access

Sequential evaluation for a Viewer-role user querying data through a Fabric shortcut pointing to ADLS Gen2. Failure at any step results in access denied.
